### ✨ lsdefine/GenericAgent (10635★)

> **一句话**：它试图用一份约 3300 行的初始“种子”代码启动智能体，让智能体在执行任务过程中自己长出技能树，逐步获得对整套系统的操作能力，同时把大模型调用的 token 开销压到传统方案的约六分之一。

- **它是什么**：这是一个用 Python 实现的“自进化”智能体项目，核心卖点不是堆很多预置工具，而是让智能体从很小的初始能力出发，在任务推进中扩展自己的技能结构。按照项目描述，它强调两件事：一是“技能树生长”，二是“更低 token 消耗下实现完整系统控制”。
  从定位上看，它更接近通用代理执行框架，而不是单纯的聊天机器人；目标是让模型不仅回答问题，还能持续学习任务模式、调用系统能力并完成较复杂的操作链路。

- **能解决什么痛点**：传统 agent 往往依赖很长的 system prompt、庞杂工具描述和频繁上下文注入，导致一跑复杂任务就非常烧 token，成本高且响应慢；这个项目显然在试图压缩这部分开销。
  另一个痛点是很多自动化 agent 只能完成预先设计好的动作，一旦遇到新任务就需要人工补工具、补 prompt；“自生长技能树”针对的就是这类泛化能力不足的问题。

- **适合谁用**：在做 AI Agent、自动化执行助手、桌面或系统级代理的 Python 开发者。
  关注“低成本长任务执行”的研究者，或者想验证 agent 自我扩展/技能沉淀机制的工程团队。

- **怎么上手**：文档未提供快速上手示例。

- **可以用在哪些场景**：让智能体接管一类多步骤系统操作任务，例如在本地环境里连续执行文件处理、命令调用和流程编排。
  用于构建需要长期执行、对 token 成本敏感的自动化代理，例如持续运行的研发辅助 agent 或运维实验 agent。
  作为研究原型，验证“少量初始代码 + 任务中技能演化”是否比传统重提示词 agent 更省上下文、更容易扩展。

- **技术看点**：项目最有辨识度的点是“从 3.3K 行 seed 生长技能树”这个设计思路，意味着它把能力扩展放在运行时演化，而不是把所有能力一开始塞进 prompt 和工具清单。
