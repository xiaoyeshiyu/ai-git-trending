✨ byJoey/cfnew (13338★)

一句话：把 Cloudflare Workers / Pages 变成一个可图形化管理的代理订阅生成端，能直接产出 VLESS、Trojan、xhttp 及多客户端配置，还自带优选 IP、延迟测试和 API 管理。

  • 它是什么：这是一个部署在 Cloudflare 平台上的订阅与节点管理方案，核心目标是用 Worker/KV 承接代理配置生成、优选 IP 管理和多客户端订阅分发。它支持 VLESS、Trojan、xhttp，多种客户端格式可直接由 Worker 生成，不再依赖外部 sub-converter，并提供图形化面板、路径级参数覆盖、ECH、ALPN、自定义 DNS 等配置能力。
  • 能解决什么痛点：一是很多人维护 Cloudflare 代理订阅时，改参数就得重新部署、手工改环境变量，这个项目通过 KV 图形化配置把“改完立即生效”做成了默认流程。二是不同客户端配置格式、优选 IP 管理、测速筛选通常要拼多个工具，这里把订阅转换、延迟测试、API 增删优选 IP 集成到同一个 Worker 里。
  • 适合谁用：需要基于 Cloudflare Workers / Pages 维护订阅分发的个人用户或小团队；熟悉 V2Ray/Xray 生态、想统一管理 Clash、Sing-box、Surge、Loon 等客户端配置的技术用户。
  • 怎么上手：文档未提供快速上手示例。README 给出的最小路径是：部署到 Cloudflare Workers 或 Pages，绑定 KV 命名空间为 C，再访问 /{你的UUID} 进入图形化配置，并将兼容性日期设置为 2026-01-20。
  • 可以用在哪些场景：为个人或团队统一生成 Clash / Sing-box / Surge 等多端订阅；在频繁更换优选 IP 的场景下，通过 API 或内置测速页动态维护节点池；给不同地区或不同客户端分发差异化连接参数时，通过 path 查询参数覆盖单节点的 ProxyIP、地区或 SOCKS5 设置。
  • 技术看点：项目把订阅转换内置到 Worker 中，直接生成 Clash / Stash / Sing-box / Surge / Loon / QuanX 配置，减少对外部转换服务的依赖。另一个值得关注的点是它对 Cloudflare KV 和转发链路做了多层优化，包括短窗口缓存、跨 isolate 版本键、WebSocket/TCP 小包合并与 fallback 降级。
