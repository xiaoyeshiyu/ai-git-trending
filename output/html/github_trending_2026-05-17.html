✨ colbymchenry/codegraph (2666★)

一句话：它先把本地代码库预处理成知识图谱，再让 Claude Code 按索引理解项目结构，从而少读文件、少发工具调用、用更少 token 完成代码问答和修改。

  • 它是什么：这是一个面向 Claude Code 的预索引代码知识图谱方案，核心思路不是让模型每次临时扫描整个仓库，而是提前把代码关系整理成可查询的结构化索引。项目强调“100% local”，也就是分析和索引过程都在本地完成，重点解决大仓库下 AI 编程助手上下文成本高、响应慢的问题。

  • 能解决什么痛点：

  • 在大型代码库里，Claude Code 为了定位函数调用链、模块依赖或相关文件，经常需要反复读文件、触发多次工具调用，既慢又烧 token。

  • 对隐私敏感的团队不希望把代码索引或分析过程交给云端服务，这类本地化方案更适合内网仓库、私有项目和受合规约束的环境。

  • 适合谁用：

  • 经常用 Claude Code 处理中大型仓库的开发者，尤其是需要跨目录追踪代码关系的人。

  • 对代码隐私有要求的团队，比如企业内部工具开发、私有业务系统维护、内网环境下的工程团队。

  • 可以用在哪些场景：

  • 接手一个陌生老项目时，先建立代码关系索引，让 Claude Code 更快回答“这个接口最终落到哪里”“某个类型被哪些模块使用”。

  • 在本地私有仓库中使用 AI 辅助编程，但又不希望把代码分析数据上传到第三方服务。
